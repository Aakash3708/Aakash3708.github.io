03 INTERNAL ENGINEERING TOOL

DawnQueue — Production Gmail Outbound Queue

PROBLEM

Professional email sent at random times in uncontrolled bursts — no scheduling discipline, no spacing enforcement, no retry safety for failed sends, and zero operational visibility.

SYSTEM BUILT

Production Gmail outbound queue built on Google Apps Script, Sheets, and Drive. Stages drafts under a HOLD label and releases them on a fully deterministic schedule — business hours, send spacing, daily caps, and exponential backoff retry.

ARCHITECTURE

13 modular source files across 4 pipeline stages: Ingest → Schedule → Send → Retry. Each stage is independently lockable, observable, and failure-safe. Idempotent sends via per-row send tokens prevent duplicate delivery.

STACK
Google Apps Script Google Sheets Google Drive Gmail API LockService Script Properties
09:45–18:00 Business Hours Enforcement (IST, Mon–Fri)
200/day Daily Send Cap with Per-Run Limits
3-tier Exponential Backoff Retry (15→30→60 min)
13 Modular Source Files — Zero Monoliths
ENGINEERING HIGHLIGHTS
  • Deterministic scheduling — scheduledSendAt is immutable once written; no queue reshuffling, no drift across trigger runs
  • LockService concurrency guards prevent duplicate sends and schedule corruption across overlapping trigger executions
  • 5m 30s execution safety cutoff with cursor-based ingest resumption — survives Apps Script's 6-minute hard limit cleanly
  • Immutable snapshot architecture — recipients, subject, HTML body, and attachments frozen to Drive at ingest time; live draft edits cannot corrupt queued sends
  • Structured Logs sheet with component, event code, and PII-redacted JSON context — full operational visibility without external tooling
